# Configurer Hyprland

Hyprland est responsable des...
- raccourcis clavier
- variables d'environnement
- affichage/moniteurs/espaces de travail
- animations de fenêtres
- ...

Référez-vous au [Wiki Hyprland](https://wiki.hypr.land/) pour les instructions de configuration générale. Seuls les détails spécifiques aux dotfiles sont documentés ci-dessous.

- `~/.config/hypr/` est le dossier de config d'Hyprland
  - Le sous-dossier `hyprland` contient les configurations par défaut
  - Le sous-dossier `custom` est pour vos configurations supplémentaires

Vous pouvez faire des changements dans le premier sous-dossier, mais ils seront écrasés si vous décidez de mettre à jour les dotfiles.

:::note
Pour remplacer les configurations par défaut, vous pourriez avoir besoin d'utiliser certaines techniques, comme `unbind` pour les raccourcis.

Et si vous n'arrivez pas à remplacer quelque chose, vous pouvez copier le fichier de configuration par défaut de `~/.config/hypr/hyprland/` vers `~/.config/hypr/custom/`, et éditer `~/.config/hypr/hyprland.conf` pour commenter la ligne qui source le fichier de configuration par défaut.

Mais si vous trouvez quelque chose qui ne va pas avec Hyprland après la mise à jour avec `install.sh`, vous devez re-vérifier `~/.config/hypr/hyprland.conf` pour voir si le fichier de configuration par défaut commenté contient des changements nécessaires.
:::

# Configurer Quickshell

Quickshell est responsable des éléments du shell comme la barre et les barres latérales

## Configuration simple

### Graphique

Appuyez sur `Super`+`I` ou pressez le bouton engrenage sur la barre latérale droite pour ouvrir les paramètres. La plupart des paramètres courants y sont. Si vous ne trouvez pas ce que vous voulez, vous pouvez essayer ce qui suit

### Éditer le fichier de configuration

Le fichier de configuration est `~/.config/illogical-impulse/config.json`

Pour l'explication des options et des valeurs par défaut, voir `~/.config/quickshell/ii/modules/common/Config.qml`

:::note
Pour les configurations qui ne sont pas limitées à Quickshell, voir la section "Divers".
:::
### Format de date et heure

Vous pouvez changer le format d'heure entre 24h et 12h dans Paramètres > Services > Temps.

Alternativement, dans le fichier de configuration, éditez `time.format`. Voir https://doc.qt.io/qt-6/qtime.html#toString pour le format.

## Configuration avancée

- Éditez les fichiers dans `~/.config/quickshell/ii`
- Voir [Documentation Quickshell](https://quickshell.outfoxxed.me/docs/types/)
  - Il est recommandé que vous parcouriez au moins les pages Introduction, Positioning, et QML Language

# Divers

## Changer l'échelle de l'interface

Pour changer l'échelle du shell, ouvrez `~/.config/quickshell/ii/shell.qml` et ajustez cette ligne
```qml
//@ pragma Env QT_SCALE_FACTOR=1
```

Pour changer l'échelle de tout sur votre écran, voir [Monitors - Wiki Hyprland](https://wiki.hypr.land/Configuring/Monitors/)

## Changer la taille de police

### Applications GTK

- Utilisez `gnome-tweaks` si vous voulez une interface graphique avec un sélecteur de police
- Sinon, vous pouvez utiliser `gsettings`
```bash
# Syntaxe
gsettings set org.gnome.desktop.interface font-name 'NOM_POLICE TAILLE_POLICE'
# Par défaut de ces dotfiles
gsettings set org.gnome.desktop.interface font-name 'Rubik 11'
```

### Applications Qt
Vous pouvez utiliser l'application Paramètres Système KDE pour personnaliser les polices

## Verrouillage/timeout d'écran

### Timeout

Référence : [Wiki Hyprland](https://wiki.hypr.land/Hypr-Ecosystem/hypridle/)

Éditez `~/.config/hypr/hypridle.conf` selon vos besoins.

### Utiliser un autre programme de verrouillage d'écran
Référence : [Arch Wiki](https://wiki.archlinux.org/title/Session_lock)

Prenons `swaylock` comme exemple.

Éditez `~/.config/hypr/hypridle.conf` et modifiez la valeur de `$lock_cmd` comme suit :
```ini
$lock_cmd = swaylock
```

Redémarrez `hypridle` après avoir fait les changements (`pkill hypridle; hypridle & disown`).

Maintenant `loginctl lock-session` appellera `swaylock` pour verrouiller l'écran.

:::note[Comment ça marche]
La commande `loginctl lock-session` enverra un signal au processus `hypridle` en cours d'exécution, puis `hypridle` déclenchera la commande de la valeur de `$lock_cmd`.
:::
