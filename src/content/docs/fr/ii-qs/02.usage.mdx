## Vue d'ensemble

- Ouvrir avec `Super` ou `Super`+`Tab`
- Glissez une fenêtre vers un autre espace de travail pour l'y envoyer
- Clic du milieu pour fermer la fenêtre

## Lanceur

- Intégré à la vue d'ensemble. Commencez juste à taper.
- Les résultats peuvent inclure :
  - Recherche d'applications
  - Résultat mathématique
  - [Préfixe : `/`] Actions du lanceur
  - [Préfixe : `;`] Historique du presse-papiers
  - [Préfixe : `:`] Emojis
  - Actions supplémentaires pour le contenu tapé : exécuter une commande, effectuer une recherche

## Contrôles de la barre

- Widget média :
  - Clic du milieu pour Lire/Pause
  - Clic droit pour Piste suivante
  - Mouse4/Mouse5 pour Piste précédente/suivante
- Luminosité : Scroll coin supérieur gauche
- Volume : Scroll coin supérieur droit
- Espaces de travail :
  - Mouse4 pour basculer l'espace de travail spécial
  - Clic droit pour ouvrir la vue d'ensemble

# Barre latérale gauche

## Contrôles généraux
- Complétion par tabulation dans les onglets Intelligence et Anime
  - Quand des suggestions apparaissent au-dessus de la zone de saisie, utilisez les flèches haut/bas pour sélectionner l'option désirée, puis appuyez sur Tab pour confirmer votre choix.
- Vous voulez emmener l'assistant avec vous ? Pour détacher la barre latérale dans une fenêtre, appuyez sur `Super`+`Alt`+`A`.

## Chat IA

- Tapez `/model` pour sélectionner un modèle. Les modèles Ollama installés localement sont détectés automatiquement.
- Tapez `/key` pour des instructions sur comment obtenir une clé API pour les modèles en ligne.
- Actuellement, les modèles Gemini avec outils peuvent éditer votre config shell. Vous pouvez, par exemple, lui dire "cache les icônes d'applications sur mes espaces de travail".

## Traducteur

Géré avec l'outil en ligne de commande `trans`. Pas grand-chose à dire ici...

## Boorus d'anime

- Tapez `/mode` et voyez les fournisseurs disponibles.
- Tapez 1 ou plusieurs tags pour voir les résultats. Les suggestions apparaissent pendant que vous tapez.
- Tapez un nombre pour sélectionner la page des résultats à voir

# Fond d'écran & couleurs
- Pour choisir un fond d'écran, appuyez sur `Ctrl`+`Super`+`T` et sélectionnez une image. Cela va aussi
  - Changer la couleur du shell, des applications Qt, et des applications GTK
  - Changer la position de l'horloge d'arrière-plan vers la région la moins encombrée de l'image (ne fonctionne actuellement bien qu'avec un seul moniteur)

# Groupes d'espaces de travail

## Navigation & Gestion des espaces de travail

Si 10 espaces de travail ne suffisent pas, vous pouvez utiliser l'ensemble suivant d'espaces de travail, jusqu'à la limite d'Hyprland. Pour commencer, faites simplement défiler le widget d'espaces de travail sur la barre vers la droite (ou utilisez les raccourcis) pour basculer vers un nouveau groupe.

Les raccourcis et widgets du shell fonctionnent de manière transparente dans le groupe d'espaces de travail focalisé. Vous n'avez rien d'extra à configurer si vous ne changez pas le nombre d'espaces de travail affichés sur l'indicateur d'espaces de travail ou la vue d'ensemble.

- **Naviguer / déplacer des espaces de travail dans un groupe** : Les raccourcis Hyprland standards s'appliquent.
  - `Super` + `2` → espace de travail **12** si vous êtes dans l'espace de travail **11-20 (groupe 2)**.
  - `Super` + `Alt` + `3` déplace silencieusement la fenêtre focalisée → espace de travail **23** si vous êtes dans l'espace de travail **21-30 (groupe 3)**.

- **Naviguer entre les groupes** :
  - Considérez ajouter les raccourcis Hyprland suivants pour sauter aux groupes d'espaces de travail adjacents
    ```ini title="~/.config/hypr/custom/keybinds.conf"
    bind = SUPER ALT, Z, workspace, r-10
    bind = SUPER ALT, X, workspace, r+10
    ```
- La **Vue d'ensemble** et **l'Indicateur d'espaces de travail** s'adaptent de manière transparente. Le contenu affiché est basé sur le groupe d'espaces de travail actuel.
  
:::tip
Si vous souhaitez modifier les raccourcis ou inclure [plus de fonctionnalités](https://wiki.hypr.land/Configuring/Dispatchers/) pour la navigation des espaces de travail, utilisez le script `~/.config/hypr/hyprland/scripts/workspace_action.sh`, au lieu de `hyprctl dispatch`. Cela va déterminer le groupe d'espaces de travail actuel en utilisant l'ID de l'espace de travail actif et dispatcher vers l'espace de travail approprié.
:::

## Considérations multi-moniteurs

Considérez ces stratégies pour une gestion multi-moniteurs efficace :
- Allouer le groupe 1 (espaces de travail 1-10) au moniteur principal tandis que le groupe 2 (espaces de travail 11-20) au moniteur secondaire :
  - Au démarrage, déplacez manuellement l'espace de travail de départ sur le moniteur secondaire dans le second groupe (par exemple, espace de travail 11) en utilisant `Super` + `0`, puis `Ctrl` + `Super` + `Right`.
  - En effet, cela va aussi créer un widget de vue d'ensemble séparé par moniteur.
- Utilisez la [liaison d'espaces de travail](https://wiki.hypr.land/Configuring/Workspace-Rules/#rules) pour toujours placer des espaces de travail spécifiques sur des moniteurs spécifiques. Obtenez tous les noms de vos moniteurs en utilisant `hyprctl monitors | grep Monitor`.
  ```ini title="~/.config/custom/general.conf"
  # Lier les espaces de travail 1-10 (groupe 1) au moniteur principal
  workspace = 1, monitor:eDP-1, default:true
  workspace = 2, monitor:eDP-1, default:true
  ...

  # Lier les espaces de travail 11-20 (groupe 2) au moniteur secondaire
  workspace = 11, monitor:HDMI-A-1, default:true
  workspace = 12, monitor:HDMI-A-1, default:true
  ...
  ```
- Focalisez sur deux espaces de travail à la fois sur chaque moniteur et déplacez/échangez les fenêtres entre moniteur/groupes (en utilisant `Super` + `Left/Right/Up/Down` ou `Super` + `Shift` + `Left/Right/Up/Down`) selon les besoins.

:::note
Les groupes d'espaces de travail ne sont pas nativement supportés par Hyprland ou tout système de barre de statut/widget. Cette config y parvient simplement en bricolant avec les dispatchers Hyprland et la config du shell.
:::
